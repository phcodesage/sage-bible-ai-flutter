Key Principles
==============

1. Feature-First Structure
   - Each feature is self-contained
   - Easy to add/remove features
   - Clear boundaries

2. Core vs Features
   - Core: Shared across features
   - Features: Isolated functionality

3. Layer Separation
   - Screens: UI layer
   - Providers: State layer
   - Services: Business logic layer
   - Models: Data layer

4. Scalability
   - Add new features in features/
   - Extend core for shared functionality
   - Keep features independent


File Responsibilities
=====================

Core Layer:
-----------
app_constants.dart      → App-wide constants (keys, durations)
user_model.dart         → User data structure
app_router.dart         → Navigation + routing logic
storage_service.dart    → Local storage wrapper
app_theme.dart          → Theme, colors, typography

Auth Feature:
-------------
auth_provider.dart      → Auth state management (Riverpod)
auth_service.dart       → Auth business logic
login_screen.dart       → Login UI
register_screen.dart    → Registration UI
auth_text_field.dart    → Reusable text input

Splash Feature:
---------------
splash_screen.dart      → Animated splash + auth check

Home Feature:
-------------
home_screen.dart        → Main home screen after login

Entry Point:
------------
main.dart               → App initialization + Riverpod setup


Dependencies Flow
=================

main.dart
  ├── Initializes StorageService
  ├── Sets up Riverpod ProviderScope
  └── Provides GoRouter
      │
      ├── Watches AuthProvider
      └── Redirects based on auth state
          │
          ├── Splash Screen
          │   └── Checks auth via AuthProvider
          │
          ├── Login/Register Screens
          │   └── Updates AuthProvider
          │
          └── Home Screen
              └── Reads AuthProvider


State Flow
==========

User Action
    ↓
AuthNotifier (Riverpod)
    ↓
AuthService
    ↓
StorageService
    ↓
AuthState Updated
    ↓
GoRouter Listens
    ↓
Navigation Triggered
    ↓
Screen Rendered


Adding New Features
===================

1. Create folder: features/your_feature/
2. Add subfolders: providers/, screens/, services/, widgets/
3. Create files following existing patterns
4. Register routes in app_router.dart
5. Create providers if needed
6. Implement UI in screens/
7. Add business logic in services/

Best Practices
==============

✓ Keep features isolated
✓ Use Riverpod for state
✓ Document all public APIs
✓ Follow naming conventions
✓ Write tests for services
✓ Keep widgets small
✓ Use const constructors
✓ Separate UI from logic
✓ Handle errors gracefully
✓ Use meaningful names
